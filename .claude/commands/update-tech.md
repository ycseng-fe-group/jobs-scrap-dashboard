현재 날짜 기준 프론트엔드 채용 트렌드를 반영하여 src/constants/techCategories.ts 의 기술스택을 업데이트합니다.

인수: $ARGUMENTS (특정 카테고리만 업데이트하려면 카테고리명 전달, 없으면 전체 업데이트)

다음 순서로 진행하세요:

---

## 1단계: 현재 날짜 및 파일 확인

현재 날짜를 확인합니다:

```bash
date "+%Y년 %m월"
```

`src/constants/techCategories.ts` 를 읽어 현재 상태를 파악합니다.

---

## 2단계: 현재 날짜 기준 트렌드 분석 및 업데이트 계획 수립

확인한 연도/월을 기준으로, 해당 시점의 프론트엔드 채용 시장 트렌드를 반영한 업데이트 계획을 수립합니다.

분석 기준:
- 현재 시점 기준 최근 6개월~1년간 채용공고에서 급증한 기술
- 현재 시점 기준 사실상 표준이 된 기술 (미등록이면 추가)
- 현재 시점 기준 쇠퇴하거나 다른 기술로 대체된 기술 (제거 또는 유지 여부 판단)
- 새로 주목받는 카테고리 신설 필요 여부

각 카테고리별 추가/제거/신설 목록을 정리하여 사용자에게 제시하고 승인을 받습니다.

---

## 3단계: 파일 업데이트

승인된 계획에 따라 `src/constants/techCategories.ts` 를 수정합니다:

- `TECH_CATEGORIES` 배열 업데이트 (기술 추가/제거/카테고리 신설)
- `TECH_ALIASES` 에 새 기술의 별칭 추가 (소문자, 점 표기, 약어 등)
- `ALL_TECHS`, `TECH_COLOR_MAP` 은 자동 파생되므로 수정 불필요

업데이트 규칙:
- 실제 채용공고에서 자주 등장하는 기술만 추가 (틈새 기술 제외)
- 각 카테고리는 최대 10개 이내로 유지
- 기존 색상 스킴 유지, 새 카테고리는 미사용 Tailwind 색상 지정
- $ARGUMENTS 가 있으면 해당 카테고리만 수정

---

## 4단계: 빌드 검증

```bash
pnpm build
```

오류가 있으면 즉시 수정합니다.

---

## 5단계: 커밋

현재 날짜를 커밋 메시지에 반영합니다:

```bash
git add src/constants/techCategories.ts
git commit -m "chore: $(date '+%Y-%m') 채용 트렌드 기반 기술스택 카테고리 업데이트"
```
