기능 개발부터 검수, 프로덕션 배포까지 전체 워크플로우를 진행합니다.

인수: $ARGUMENTS (구현할 기능 설명)

다음 순서로 진행하세요:

---

## 1단계: 기능 분석 및 계획

$ARGUMENTS 를 바탕으로 아래를 파악합니다:

- 수정/생성이 필요한 파일 목록
- 구현 접근 방식 요약 (1~3줄)

계획을 간략히 사용자에게 제시하고 승인을 받은 후 2단계로 진행합니다.

---

## 2단계: 기능 구현

승인된 계획에 따라 코드를 작성합니다:

- React/TypeScript 컨벤션 준수
- Tailwind CSS 클래스 사용
- 기존 코드 스타일 유지
- 불필요한 abstraction 지양 (현재 요구사항에 맞게만 구현)

---

## 3단계: 코드 검수 (simplify 기준 적용)

구현한 파일들을 아래 기준으로 검토하고 문제가 있으면 즉시 수정합니다:

**React/TypeScript**
- 중복 코드 또는 재사용 가능한 로직 분리 여부
- 불필요한 리렌더링 (useMemo, useCallback 누락 또는 과도한 사용)
- 타입 정의 누락 또는 `any` 남용
- props drilling 이 깊은 경우 컨텍스트 또는 상태 관리 제안

**Tailwind CSS**
- 반복되는 클래스 조합 → 공통 컴포넌트나 `cn()` 유틸로 추출
- 반응형 클래스 누락

**일반**
- 사용하지 않는 import, 변수, 함수 제거
- 복잡한 조건식 단순화

검수 결과를 한 줄로 요약해서 사용자에게 알립니다.

---

## 4단계: 빌드 검증

```bash
pnpm build
```

빌드 오류가 있으면 즉시 수정 후 재빌드합니다. 성공하면 다음 단계로 진행합니다.

---

## 5단계: 커밋

변경 파일을 스테이징하고 커밋합니다 (`.env`, `data/`, `.venv/` 제외):

```bash
git add <관련 파일>
git commit -m "<컨벤션에 맞는 커밋 메시지>"
```

커밋 메시지 컨벤션:
- `feat:` 새 기능
- `fix:` 버그 수정
- `refactor:` 리팩토링
- `chore:` 설정·의존성·스크립트 변경

---

## 6단계: 기술스택 업데이트

배포 전에 현재 날짜를 확인하고 `src/constants/techCategories.ts` 를 최신 채용 트렌드에 맞게 업데이트합니다:

```bash
date "+%Y년 %m월"
```

`src/constants/techCategories.ts` 를 읽고, 현재 날짜 기준으로 아래를 검토합니다:
- 최근 채용공고에서 급증한 신규 기술 (추가 여부)
- 사실상 대체된 기술 (제거 또는 유지 여부)
- 새로 주목받는 카테고리 신설 필요 여부

변경이 필요하면 사용자에게 한 줄로 요약 후 승인을 받고 수정합니다.
변경이 없으면 "기술스택 최신 상태 확인 완료" 라고 알리고 넘어갑니다.

변경된 경우 커밋합니다:

```bash
git add src/constants/techCategories.ts
git commit -m "chore: $(date '+%Y-%m') 채용 트렌드 기반 기술스택 카테고리 업데이트"
```

---

## 7단계: 프로덕션 배포

사용자에게 "프로덕션 배포를 진행할까요?" 라고 확인을 받은 후 배포합니다:

```bash
vercel --prod
```

배포가 완료되면 배포 URL을 사용자에게 알려줍니다.
