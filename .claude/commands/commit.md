변경된 파일을 검토하고 커밋을 생성합니다.

인수: $ARGUMENTS (커밋 메시지 힌트, 없으면 변경 내용을 보고 자동 생성)

다음 순서로 진행하세요:

1. 변경 사항을 파악합니다:

```bash
git status
git diff
```

2. 변경 내용을 분석하여 아래 컨벤션에 맞는 커밋 메시지를 작성합니다:

   - `feat:` 새 기능
   - `fix:` 버그 수정
   - `refactor:` 리팩토링
   - `chore:` 설정·의존성·스크립트 변경
   - `docs:` 문서 변경

3. 관련 파일만 스테이징하고 커밋합니다 (`.env`, `data/`, `.venv/` 제외):

```bash
git add <관련 파일>
git commit -m "<생성한 커밋 메시지>"
```

$ARGUMENTS 가 있으면 해당 내용을 커밋 메시지에 반영하세요.
