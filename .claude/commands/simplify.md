변경된 코드를 검토하고 품질·효율성을 개선합니다.

인수: $ARGUMENTS (특정 파일 경로나 컴포넌트 이름, 없으면 최근 변경 파일 전체 검토)

다음 순서로 진행하세요:

1. 검토 대상 파일을 파악합니다:
   - $ARGUMENTS 가 있으면 해당 파일을 검토합니다.
   - 없으면 `git diff --name-only` 로 변경된 파일을 확인합니다.

2. 각 파일을 읽고 아래 기준으로 검토합니다:

   **React/TypeScript 기준**
   - 중복 코드 또는 재사용 가능한 로직의 컴포넌트/훅 분리 여부
   - 불필요한 리렌더링 (useMemo, useCallback 누락 또는 과도한 사용)
   - 타입 정의 누락 또는 `any` 남용
   - props drilling 이 깊은 경우 컨텍스트 또는 상태 관리 제안

   **Tailwind CSS 기준**
   - 반복되는 클래스 조합 → 공통 컴포넌트나 `cn()` 유틸로 추출
   - 반응형 클래스 누락

   **일반 기준**
   - 사용하지 않는 import, 변수, 함수 제거
   - 복잡한 조건식 단순화
   - 명확하지 않은 변수명 개선

3. 문제를 발견하면 직접 수정합니다. 수정이 크면 먼저 요약을 제시하고 확인 후 진행합니다.
